# Security policy

## Supported versions

Security fixes land on `main` and tagged releases. Only the latest minor release line may receive backports unless noted in release notes.

## Reporting a vulnerability

Do not file public issues for undisclosed vulnerabilities.

Use **GitHub** → **Security** → **Advisories** → **Report a vulnerability**, or contact repository maintainers privately.

Include impact, reproduction steps, and affected packages (contracts, backend, frontend).

## Safe harbor

We welcome coordinated disclosure and good-faith research. Do not disrupt production systems or access data without authorization.
